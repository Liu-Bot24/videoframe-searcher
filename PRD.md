# 产品需求文档 (PRD)：VideoFrame Searcher (视频截图检索引擎) v2.1

## 1. 产品概述
* **产品名称：** VideoFrame Searcher
* **运行平台：** Windows OS (Windows 10/11)
* **产品定位：** 一款集视频精准下载、自定义抽帧、大容量图片管理与 Google 自动化反向搜图于一体的桌面端效率工具。
* **核心价值：** 帮助用户将流媒体视频快速转化为静态切片，并自动化执行批量的反向图像搜索。

## 2. 核心技术栈建议
* **GUI 框架：** Python + PyQt5 / PySide6 (需采用多线程架构防止界面卡死)。
* **视频下载：** `yt-dlp` (需支持独立更新)。
* **视频处理：** `FFmpeg` (高效本地抽帧)。
* **自动化搜图：** Playwright 或 Selenium (用于处理跨域、绕过 API 限制的自动化本地图片上传)。

---

## 3. 核心功能与业务逻辑

### 3.1 视频下载模块
* **输入与解析：** 用户输入目标 URL，程序解析视频元数据（标题、时长、是否直播）。
* **下载规则限制：**
  * **拦截直播流：** 若检测到 `is_live=True`，阻断下载并提示“暂不支持直播流提取”。
  * **画质与格式精简：** 仅下载视频流（忽略音频），最高限制 1080p，节约带宽与空间。
  * **权限突破：** 设置中提供 `[开启本地浏览器 Cookie 授权]` 选项，以支持需要登录的视频下载。

### 3.2 截图/抽帧模块
* **参数设置：**
  * 自定义抽帧间隔（如：每 5 秒 1 张）。
  * **[复选框]** `重新截图时，清除该视频原有的截图文件`。
* **边界保护：**
  * 最小间隔限制为 1 秒。
  * **容量预警：** 执行前计算 `视频总时长 / 间隔`，若预估生成截图超过 1000 张，弹窗二次确认以防撑爆硬盘或内存。

### 3.3 图片浏览与项目管理
* **项目创建与命名：**
  * 自动创建工作区子目录，格式：`YYYYMMDD_HHMMSS_视频合法标题`。
* **历史回溯：** 左侧边栏按时间倒序展示历史项目，点击即可重新加载图片和搜索进度。
* **画廊视图与性能优化：**
  * 以网格展示截图，支持单选、多选、全选。
  * **强制要求：** 必须实现 UI 懒加载 (Lazy Load) 或分页，绝对禁止一次性将数千张原图读入内存，防止 OOM 崩溃。
* **垃圾回收：** 提供 `[删除原始视频，仅保留截图]` 的磁盘清理功能。

### 3.4 自动化 Google 以图搜图模块
* **执行逻辑：** 获取选中的多张本地图片路径，后台唤醒自动化浏览器（如 Playwright），模拟真实用户向 Google Lens 表单依次上传图片。获取到搜索结果页 URL 后，调用系统默认浏览器打开。
* **并发与风控限制：**
  * **单次上限：** 单次最多允许执行 10 张图片的搜索（超出则置灰按钮或弹窗拦截），防止浏览器标签页风暴。
  * **队列机制：** 多张图片必须串行上传，请求间隔加入 2~5 秒随机延迟，降低触发 Google reCAPTCHA 验证码的概率。

## 4. 👨‍💻 给 AI 开发者的专属备注 (高危风险点与架构要求)

请在编写代码时，务必注意并解决以下技术暗坑：

1. **Windows 长路径限制 (MAX_PATH Issue)：**
   * **风险：** 视频标题过长可能导致生成的文件路径超过 Windows 默认的 260 字符限制，导致 FFmpeg 写入失败。
   * **要求：** 在拼接项目文件夹名称时，对视频标题进行强制截断（例如最多保留前 20 个合法字符），并过滤掉所有 Windows 非法路径字符 (`\/:*?"<>|`)。

2. **僵尸进程与生命周期管理 (Zombie Processes)：**
   * **风险：** GUI 异常关闭或被强杀时，后台运行的 yt-dlp、FFmpeg 或无头浏览器会变成僵尸进程，持续消耗资源并锁死文件。
   * **要求：** 必须监听主程序的 Exit Event。在退出前，确保发送终止信号强杀所有派生的子进程。

3. **FFmpeg 抽帧性能优化 (禁止使用循环调用的笨办法)：**
   * **要求：** 严禁使用 for 循环结合 `-ss` 参数反复调用 FFmpeg，这会导致极低的效率和不精准的截图。必须使用 FFmpeg 的 `fps` 视频滤镜。例如每 5 秒一帧的单次调用命令：`ffmpeg -i input.mp4 -vf fps=1/5 screenshots/out%04d.jpg`。

4. **yt-dlp 的保质期与动态更新：**
   * **风险：** 视频平台反爬策略更新极快，内置的 yt-dlp 容易失效。
   * **要求：** 必须在软件设置界面提供“强制更新下载核心”按钮，或在后台静默执行 `yt-dlp -U`。

5. **Google Lens 自动化 DOM 腐烂 (Selector Rot) 与代理：**
   * **风险：** Google 前端结构变化会导致 Playwright 找不到上传按钮。高频请求会导致 IP 被封。
   * **要求：** 元素定位尽量鲁棒（使用文本或 Aria 标签），对 `TimeoutError` 做捕获并提示用户。设置中需预留“网络代理 (HTTP Proxy)”配置项，以备 IP 风控时使用。
